% ── Abstract ──────────────────────────────────────────────────────────────
\begin{abstract}
PhotoMind is a multimodal personal photo knowledge retrieval system that transforms a user's photo library into a queryable knowledge base. The system uses four specialized CrewAI agents---Controller, Photo Analyst, Knowledge Retriever, and Insight Synthesizer---orchestrated via hierarchical delegation to answer natural-language queries over personal photos (receipts, food, documents, screenshots). We extend the base system with two reinforcement learning components: (1)~a contextual bandit (Thompson Sampling, UCB1, $\varepsilon$-greedy) that routes queries across four search strategies (factual, semantic, behavioral, embedding), and (2)~a Deep Q-Network (DQN) confidence calibrator with a non-terminal \textit{requery} action that enables multi-step evidence gathering. Both components are trained offline via a deterministic simulator at \$0 API cost. A 7-configuration ablation study across 5 random seeds with 95\% confidence intervals demonstrates that the DQN eliminates silent failures (confident-but-wrong answers) from 2.4\% to 0.0\% on training queries, with the Recommended (Rule+DQN) configuration achieving 0.4\% silent failure rate while preserving baseline routing accuracy. The system includes a React frontend with FastAPI backend, dense vector embedding search via sentence-transformers (all-MiniLM-L6-v2), and comprehensive evaluation infrastructure with 56 queries, statistical significance testing, and held-out generalization validation.
\end{abstract}

\section{Introduction}
\label{sec:introduction}

Smartphone users accumulate thousands of photos---receipts, bills, food, screenshots, documents---but cannot search them by meaning or extract facts using natural language. PhotoMind addresses this gap by building a structured knowledge base from photo analysis and providing confidence-graded answers with source attribution.

\subsection{Core Capabilities}

\begin{itemize}[nosep]
  \item Analyze photos using GPT-4o Vision to extract text (OCR), entities, and semantic descriptions
  \item Build a persistent structured knowledge base from the analysis results
  \item Answer natural-language queries with four distinct retrieval strategies
  \item Return confidence-graded answers (A--F) with source photo attribution
  \item Gracefully decline queries the knowledge base cannot answer
\end{itemize}

\subsection{Real-World Use Cases}

\begin{itemize}[nosep]
  \item \textit{``How much did I spend at ALDI?''} --- finds all 5 ALDI receipts, aggregates total with source photos
  \item \textit{``Show me photos of pizza''} --- finds food photos matching the description
  \item \textit{``What type of food do I photograph most?''} --- analyzes patterns across all 25 photos
  \item \textit{``What was my electric bill?''} --- correctly declines (no such photo in the library)
\end{itemize}

\subsection{Project Scope and Contributions}

This project makes four principal contributions:

\begin{enumerate}[nosep]
  \item \textbf{Multi-component ablation methodology} that isolates individual and interaction effects of modular RL components, revealing that upstream stochasticity compounds into downstream variance.
  \item \textbf{Zero-cost offline training} via deterministic simulation---a transferable pattern for any agentic system with expensive API calls.
  \item \textbf{Asymmetric reward design} that prioritizes safety over coverage in personal-data domains.
  \item \textbf{Requery as a non-terminal MDP action}, transforming single-step confidence classification into multi-step active information gathering with structurally relevant discounting.
\end{enumerate}

\section{Agent Design}
\label{sec:agents}

PhotoMind uses four specialized agents across two crews.

\subsection{Controller Agent (Query Crew Manager)}

The Controller acts as the chief librarian: it classifies query intent (factual, semantic, behavioral) and delegates to the appropriate specialist agent. It operates as the manager in CrewAI's hierarchical process with \texttt{allow\_delegation=True}.

\textit{Backstory: ``You are the chief librarian of a personal photo knowledge base. You classify first, then delegate. If you are unsure, you say so.''}

\subsection{Photo Analyst (Ingestion Crew)}

Scans the \texttt{photos/} directory and passes each image to GPT-4o Vision for structured extraction: image classification, OCR text, semantic description, and named entities (amounts, dates, vendors, locations). Reports partial extraction with lower confidence rather than guessing on unclear images.

\subsection{Knowledge Retriever}

In ingestion: writes the knowledge base JSON. In queries: executes the search using the RL-enhanced \texttt{PhotoKnowledgeBaseTool}. The query task restricts this agent to only the primary search tool via task-level tool override, preventing fallback searches that produce misleading weak matches.

\subsection{Insight Synthesizer}

Synthesizes grounded, human-readable answers with three strict rules: (1)~every claim cites a source photo, (2)~every answer includes a confidence grade A--F, (3)~ambiguous evidence is explicitly flagged.

\subsection{Agent Collaboration Pattern}

In the ingestion phase (sequential), each task's output becomes the next task's input via CrewAI's \texttt{context} parameter. In the query phase (hierarchical), the Controller delegates to the Retriever, whose structured JSON output (\texttt{confidence\_grade}, \texttt{confidence\_score}, \texttt{source\_photos}, \texttt{answer\_summary}) flows to the Synthesizer for final answer generation.

\section{Search Strategies}
\label{sec:search-strategies}

The knowledge base tool implements four retrieval strategies, each optimized for a different query type. The contextual bandit (Section~\ref{sec:bandit}) learns to route queries to the optimal strategy.

\subsection{Factual Search}

Targets receipts, bills, and documents with specific extractable facts:
\begin{enumerate}[nosep]
  \item Match query words against structured entities (vendor names, amounts, dates) --- each match scores $+0.4$
  \item Match against OCR text (keyword frequency, stop words excluded) --- scores up to $+0.5$
  \item Image-type context boost: $+0.2$
  \item Aggregation mode for ``how much'' / ``total'' queries returns all matching results with computed totals
\end{enumerate}

\subsection{Semantic Search}

Targets photos by visual meaning:
\begin{enumerate}[nosep]
  \item Apply text cleaning (lowercase, strip punctuation) to query and descriptions
  \item Compute overlap of meaningful words (length $> 3$) between query and description
  \item Score: $\frac{|\text{meaningful overlap}|}{\max(|\text{meaningful query words}|, 1)} \times 0.8$
  \item Image-type boost: $+0.2$
\end{enumerate}

\subsection{Behavioral Search}

Targets corpus-wide pattern questions:
\begin{enumerate}[nosep]
  \item Aggregate \texttt{image\_type} distribution across all photos
  \item Aggregate entity value frequencies (most common vendors, items)
  \item Return representative photos from the dominant pattern with full distribution stats
\end{enumerate}

\subsection{Embedding Search (New)}

Dense vector retrieval using sentence-transformers, added as the 4th bandit arm:
\begin{enumerate}[nosep]
  \item Encode all photo text (description + OCR + caption + entities) using \texttt{all-MiniLM-L6-v2} (384-dimensional vectors)
  \item Encode the query with the same model
  \item Rank by cosine similarity: $\text{score}(q, p) = \frac{\mathbf{e}_q \cdot \mathbf{e}_p}{\|\mathbf{e}_q\| \|\mathbf{e}_p\|}$
  \item Embeddings cached to \texttt{knowledge\_base/embeddings.npz} for instant reload
  \item Graceful fallback to semantic search if sentence-transformers unavailable
\end{enumerate}

This strategy runs entirely locally with zero API cost, providing true dense semantic retrieval that captures meaning beyond keyword overlap.

\end{lstlisting}

\subsection{Query Crew (Hierarchical with Planning)}

\begin{lstlisting}[language={},caption={Query pipeline with hierarchical delegation.}]
Manager: Controller Agent
  |-- Task 1 -> Knowledge Retriever: search KB with PhotoKnowledgeBaseTool
  +-- Task 2 -> Insight Synthesizer: synthesize answer    [context: Task 1]
\end{lstlisting}

Key crew settings: \texttt{process=Process.hierarchical}, \texttt{manager\_agent=controller}, \texttt{planning=True}, \texttt{memory=True}.

\subsection{Feedback Loop}

The \texttt{FeedbackStore} implements persistent adaptive confidence thresholds:
\begin{itemize}[nosep]
  \item After each evaluation query, outcome is recorded (correct/incorrect, strategy, confidence score)
  \item Per-strategy accuracy rates are computed once $\geq 3$ samples exist
  \item Strategies with accuracy $< 70\%$ get $+0.05$ threshold increase (more conservative)
  \item Strategies with accuracy $\geq 90\%$ get $-0.05$ threshold decrease (less conservative)
\end{itemize}

\subsection{Communication Protocols}

Agents communicate through four mechanisms:
\begin{enumerate}[nosep]
  \item \textbf{Context passing} --- CrewAI's \texttt{context=[previous\_task]} for structured data flow
  \item \textbf{Hierarchical manager delegation} --- Controller oversees all delegation
  \item \textbf{Task-level tool restriction} --- prevents fallback tools that produce weak matches
  \item \textbf{Structured tool output} --- standardized JSON schema acts as agent-to-agent protocol
\end{enumerate}

\section{Reinforcement Learning Extension}
\label{sec:rl}

\subsection{Motivation}

The base system's rule-based \texttt{\_classify\_query()} achieves 100\% routing accuracy on the original 20 test cases but only 76.8\% on the expanded 56-query set including 11 deliberately ambiguous queries. Additionally, 1.8\% of queries result in \textbf{silent failures}---the most dangerous failure mode for a personal data system.

\textbf{Design choice: two RL components.} Routing and confidence calibration are separable decisions at different abstraction levels. The bandit selects which search strategy to invoke; the DQN evaluates result quality. Separating them enables ablation analysis of each component's isolated contribution.

\subsection{RL Approach 1: Contextual Bandits}
\label{sec:bandit}

\subsubsection{Problem Formulation}

A $K$-arm contextual bandit where arms correspond to search strategies: $\mathcal{A} = \{\text{factual},\; \text{semantic},\; \text{behavioral},\; \text{embedding}\}$, with context derived from query features.

\subsubsection{Context Clustering}

Context clusters are computed via $K$-Means on the feature space:
\begin{equation}
  c = \arg\min_{k=1}^{K} \| \varphi(q) - \boldsymbol{\mu}_k \|^2
\end{equation}
where $\varphi: \mathcal{Q} \to \mathbb{R}^{12}$ is the feature extractor and $\boldsymbol{\mu}_k$ are KMeans centroids ($K=4$).

\subsubsection{Feature Extractor}

The query feature vector $\varphi(q) \in \mathbb{R}^{12}$:
\begin{equation}
  \varphi(q) = \begin{bmatrix}
    \text{len}(q)/100 \\
    |\text{words}(q)|/20 \\
    \mathbb{1}[\text{has\_amount}(q)] \\
    \mathbb{1}[\text{has\_date}(q)] \\
    \mathbb{1}[\text{has\_vendor}(q)] \\
    \mathbb{1}[\text{has\_behavioral}(q)] \\
    \mathbb{1}[\text{has\_semantic}(q)] \\
    \mathbb{1}[\text{has\_negation}(q)] \\
    \mathbb{1}[\text{is\_wh\_question}(q)] \\
    \mathbb{1}[\text{is\_yes\_no}(q)] \\
    \mathbb{1}[\text{is\_imperative}(q)] \\
    \mathbb{1}[\text{has\_vendor\_match}(q)]
  \end{bmatrix}
\end{equation}

\subsubsection{Cluster Count Selection ($k=4$)}

Evaluated $k \in \{2,3,4,5,6,8\}$ on the 12-dimensional feature space (560 samples after 10$\times$ augmentation) using silhouette score:

\begin{table}[H]
\centering
\begin{tabular}{ccp{7cm}}
\toprule
$k$ & Silhouette & Interpretation \\
\midrule
2 & 0.31 & Merges ambiguous into factual/semantic \\
3 & 0.35 & No separate ambiguous cluster \\
\textbf{4} & \textbf{0.38} & Best score; clean four-way partition \\
5 & 0.36 & Splits factual without meaningful distinction \\
6 & 0.33 & Fragments clusters \\
8 & 0.28 & Over-fragmented \\
\bottomrule
\end{tabular}
\caption{Silhouette analysis for cluster count selection.}
\label{tab:silhouette}
\end{table}

The $k=4$ partition produces: Cluster~0 (factual-keyword), Cluster~1 (semantic-description), Cluster~2 (behavioral-aggregation), Cluster~3 (ambiguous/cross-category).

\subsubsection{Thompson Sampling}

Maintain Beta posteriors per (cluster $c$, arm $a$):
\begin{equation}
  P(\theta_{c,a}) = \text{Beta}(\alpha_{c,a},\; \beta_{c,a}), \quad \alpha_{c,a} = \beta_{c,a} = 1 \text{ (uniform prior)}
\end{equation}

Arm selection via posterior sampling:
\begin{align}
  \tilde{\theta}_{c,a} &\sim \text{Beta}(\alpha_{c,a},\; \beta_{c,a}) \quad \forall\, a \in \mathcal{A} \\
  a^* &= \arg\max_a \tilde{\theta}_{c,a}
\end{align}

Posterior update (binary reward, threshold $= 0.5$):
\begin{equation}
  \text{if } r > 0.5: \; \alpha_{c,a^*} \mathrel{+}= 1, \quad \text{else: } \beta_{c,a^*} \mathrel{+}= 1
\end{equation}

\subsubsection{UCB1}

UCB score per arm:
\begin{equation}
  \text{UCB}(c, a) = \bar{Q}(c, a) + C \sqrt{\frac{\ln N_c}{N_{c,a}}}
\end{equation}
where $\bar{Q}(c,a)$ is the running mean reward, $N_c = \sum_a N_{c,a}$ is incremented only after all arms have been explored, and $C = 2.0$.

\subsubsection{Bandit Reward Function}

\begin{equation}
  R_{\text{bandit}}(a, y) = \begin{cases}
    1.0 & \text{if strategy matches expected type AND photo found} \\
    0.5 & \text{if photo found but strategy mismatches} \\
    0.3 & \text{aggregate query, strategy matches} \\
    0.0 & \text{otherwise}
  \end{cases}
\end{equation}

\subsection{RL Approach 2: DQN Confidence Calibrator}
\label{sec:dqn}

\subsubsection{MDP Formulation}

\begin{itemize}[nosep]
  \item \textbf{State space} $\mathcal{S} \subseteq \mathbb{R}^8$:
    $s = [\text{top\_score},\; \text{score\_gap},\; \text{num\_results\_norm},\; \text{strategy\_idx},\; \text{query\_len},\; \text{entity\_match},\; \text{type\_match},\; \text{avg\_score}]$
  \item \textbf{Action space} $\mathcal{A} = \{\text{accept\_high},\; \text{accept\_moderate},\; \text{hedge},\; \text{requery},\; \text{decline}\}$
  \item \textbf{Discount factor} $\gamma = 0.99$
  \item \textbf{Episode length}: 1 to $(\text{MAX\_REQUERY\_STEPS} + 1) = 3$ steps
\end{itemize}

Four actions are \textbf{terminal} ($\text{done} = \text{True}$). The \textbf{requery} action is \textbf{non-terminal}: the agent incurs a step cost of $-0.1$, an alternate search strategy is selected, and the agent observes new retrieval results before acting again.

\subsubsection{Multi-Step Bellman Equation}

For terminal actions $a \in \{\text{accept\_high},\; \text{accept\_moderate},\; \text{hedge},\; \text{decline}\}$:
\begin{equation}
  Q^*(s, a) = R(s, a)
\end{equation}

For the requery action:
\begin{equation}
  Q^*(s, \text{requery}) = R(s, \text{requery}) + \gamma \cdot \max_{a'} Q^*(s', a')
  \label{eq:bellman-requery}
\end{equation}
where $s' = \text{ConfidenceState}(\text{results}_{\text{alt}},\; \text{strategy}_{\text{alt}},\; \varphi(q))$ and $\text{strategy}_{\text{alt}}$ is sampled uniformly from strategies $\neq$ current.

This makes $\gamma$ \textbf{structurally relevant}: the agent must weigh the immediate requery cost against the discounted expected value of better results from an alternate strategy.

\subsubsection{Q-Network Architecture}

\begin{equation}
  Q(s, a; \theta) = \text{FC}_3(\text{ReLU}(\text{FC}_2(\text{ReLU}(\text{FC}_1(s)))))
\end{equation}
with $\text{FC}_1: \mathbb{R}^8 \to \mathbb{R}^{64}$, $\text{FC}_2: \mathbb{R}^{64} \to \mathbb{R}^{64}$, $\text{FC}_3: \mathbb{R}^{64} \to \mathbb{R}^5$.

\textbf{Architecture transfer from LunarLander.} The FC(8$\to$64$\to$64$\to$$N$) topology transfers well because both domains share: (1) low-dimensional continuous state (8-dim), (2) small discrete action space (4--5 actions), (3) non-linear decision boundaries over compact feature vectors. Hidden size 64 was validated via sweep over $\{32, 64, 128\}$ (best reward/variance trade-off at 64).

\subsubsection{TD Learning}

\begin{equation}
  y = r + \gamma \cdot (1 - \text{done}) \cdot \max_{a'} Q(s', a'; \theta_{\text{target}})
\end{equation}

\textbf{Loss:}
\begin{equation}
  L(\theta) = \mathbb{E}_{(s,a,r) \sim \mathcal{D}} \left[ (y - Q(s, a; \theta))^2 \right]
\end{equation}

\textbf{Soft update:}
\begin{equation}
  \theta_{\text{target}} \leftarrow \tau \cdot \theta_{\text{online}} + (1 - \tau) \cdot \theta_{\text{target}}, \quad \tau = 0.001
\end{equation}

\subsubsection{Reward Matrix}

The reward matrix encodes an asymmetric safety-first design:

accept\_moderate (1) & $+0.7$ & $-0.5$ & $-0.7$ & $-0.7$ \\

\subsection{Offline Simulation Environment}

Training on live API calls would cost ${\sim}\$11{,}200$ (560{,}000 steps $\times$ \$0.02/query). Instead, \texttt{PhotoMindSimulator} pre-computes all 4 search strategies on all 56 queries once (pure Python), caches them, and serves the cache during training. \textbf{Training cost: \$0.}

Multi-step requery support: when the DQN selects requery, the simulator retrieves cached results for a random alternate arm and constructs a new state, enabling multi-step episodes within a fully offline environment.

\textbf{Query augmentation} ($10\times$ factor): synonym substitution and entity swapping expand 56 queries to $\sim$560 training samples.

\section{Discussion}
\label{sec:discussion}

\subsection{When RL Helps}

The DQN's value is clearest on queries where retrieval confidence signals are ambiguous---specifically, queries with moderate relevance scores (0.2--0.5) where the static threshold system commits to a grade while the DQN learns to hedge, requery, or decline. All 11 should-decline queries in the expanded test set receive correct decline decisions from the trained DQN.

\subsection{Multi-Step Requery Dynamics}

The requery action gives the DQN an active information-gathering capability. The $-0.1$ step cost prevents degenerate looping: the agent learns to requery only when the expected improvement outweighs the cost. This makes $\gamma = 0.99$ structurally relevant---the DQN performs genuine temporal credit assignment across up to 3 decision steps.

Requery usage follows a characteristic pattern: it spikes during mid-training (episodes 300--800) then drops to $<5\%$ in the final 200 episodes, invoked only on states with genuinely ambiguous evidence (relevance 0.25--0.35).

\subsection{Bandit $\times$ DQN Interaction Effects}

The ablation reveals two distinct interaction patterns:

\begin{enumerate}
  \item \textbf{Complementary (Thompson + DQN):} Thompson Sampling converges to stable posteriors with low variance (71.0\% $\pm$ 1.3\% routing). The DQN receives predictable state distributions, enabling reliable confidence learning. Result: 0.0\% silent failures.

  \item \textbf{Compounding failure (UCB + DQN):} UCB's aggressive exploration produces high routing variance (47.6\% $\pm$ 9.1\%), creating state distribution shift that the DQN cannot calibrate against. Result: 3.8\% silent failures---\textit{worse} than baseline.

  \item \textbf{Independence (Rule + DQN):} Deterministic routing provides stable state distributions, eliminating interaction risk while preserving DQN safety benefits. This is analogous to the ``frozen encoder'' pattern in deep learning.
\end{enumerate}

\textbf{Design lesson:} In modular RL systems, compose components only when each component's output variance is low enough for downstream components to train against.

\subsection{Routing Accuracy Regression Analysis}

Of the 56 evaluation queries, 45 have unambiguous ground-truth routing labels. Manual inspection of the 11 bandit ``disagreements'' reveals:
\begin{itemize}[nosep]
  \item \textbf{True errors (4 queries):} KMeans clustering grouped ambiguous feature vectors together
  \item \textbf{Debatable labels (7 queries):} Bandit routes to an alternative strategy that also returns correct results
\end{itemize}

The true routing error rate is $\sim$7\% (4/56), not $\sim$29\% (16/56). The remaining discrepancies reflect label ambiguity, not policy failure.

\subsection{Why Rule+DQN Outperforms Full RL}

The bandit introduces \textbf{state distribution shift}: when routing differs from rule-based, the DQN's state features (\texttt{strategy\_idx}, \texttt{type\_matches\_strategy}) shift, and the DQN sees states encountered less frequently during training. For 45 unambiguous queries this variance is pure noise; for 11 ambiguous queries it is productive. Since $45 > 11$, the net effect is negative.

\subsection{Cost Analysis}

\$0 training cost vs.\ $\sim$\$11{,}200 if trained on live API calls. The offline simulation approach is the key enabling decision.

\section{Challenges and Solutions}
\label{sec:challenges}

\begin{enumerate}
  \item \textbf{Python version incompatibility.} CrewAI requires Python $< 3.14$; macOS default was 3.14.x. \textit{Solution:} \texttt{pyenv} with Python 3.10.14.

  \item \textbf{HEIC image support.} 14 of 25 photos were iPhone HEIC format. \textit{Solution:} \texttt{pillow-heif} with \texttt{register\_heif\_opener()} and RGB mode conversion guard.

  \item \textbf{Gemini API quota exhaustion.} Free-tier 20 RPD limit exhausted during debugging. \textit{Solution:} Switched to OpenAI GPT-4o with direct ingestion script (1 call/photo).

  \item \textbf{OpenAI embeddings API blocked.} \texttt{memory=True} defaults to OpenAI embeddings. \textit{Solution:} Local sentence-transformers (\texttt{all-MiniLM-L6-v2}), zero API calls.

  \item \textbf{Punctuation breaking query matching.} ``ALDI?'' didn't match ``aldi''. \textit{Solution:} \texttt{\_clean()} helper with regex punctuation stripping.

  \item \textbf{Semantic scoring too strict.} Stop words inflated denominator. \textit{Solution:} Normalize by meaningful word count only (length $> 3$).

  \item \textbf{Evaluation case-sensitivity bug.} Mixed-case filenames caused false negatives. \textit{Solution:} Case-insensitive comparison.

  \item \textbf{Context accumulation causing TPM errors.} Reusing one Crew for 20 queries exceeded 30K TPM limit. \textit{Solution:} Fresh Crew instance per query.

  \item \textbf{Stop words causing false factual matches.} ``what'' and ``this'' matched unrelated OCR text. \textit{Solution:} Explicit \texttt{\_STOP\_WORDS} exclusion set.

  \item \textbf{Agents ignoring tool decline signals.} Agents used fallback tools for irrelevant weak matches. \textit{Solution:} Task-level tool override restricting to \texttt{PhotoKnowledgeBaseTool} only.

  \item \textbf{Aggregation truncated by top\_k.} ``How much at ALDI?'' returned 3 of 5 receipts. \textit{Solution:} Aggregation detection returns all matching results with computed totals.
\end{enumerate}

\section{Threats to Validity}
\label{sec:threats}

\subsection{Internal Validity}

\textbf{Data leakage prevention.} The 56-query evaluation suite uses a strict 42-train / 14-held-out split. RL agents never see held-out queries during training.

\textbf{Deterministic simulator.} \texttt{PhotoMindSimulator} pre-computes all search results for perfect reproducibility. To broaden the training distribution, we apply $10\times$ query augmentation (synonym substitution + entity swapping), producing 560 unique training episodes per seed.

\textbf{Reward shaping transparency.} The full reward matrix is published (Table~\ref{tab:reward-matrix}); alternative weightings would produce different policies.

\subsection{External Validity}

\textbf{Single-user knowledge base.} Results are from one user's 25 iPhone photos. The RL architecture is domain-agnostic, but learned parameters would require retraining on different photo distributions.

\textbf{LLM dependency.} Base retrieval quality depends on GPT-4o Vision's OCR and entity extraction. The offline simulator decouples RL training from the LLM.

\subsection{Construct Validity}

\textbf{Retrieval accuracy definition.} Binary metric (expected photo appears anywhere in results) prioritizes recall; rank-aware metrics (MRR, NDCG) would capture ranking quality.

\textbf{Confidence grade discretization.} The DQN's 5-action space maps to letter grades. The requery action mitigates over-commitment by gathering additional evidence.

\subsection{Statistical Conclusion Validity}

\textbf{Seed-based CIs.} With 5 seeds, 95\% CIs use the $t$-distribution (4~df). We report exact $p$-values, Cohen's $d$, and significance stars.

\textbf{Ceiling effects.} Several metrics are near 100\%, compressing the space for RL improvement. Silent failure rate---the safety-critical metric---shows the clearest benefit (2.4\% $\to$ 0.0\%).

\section{Ethical Considerations}
\label{sec:ethics}

\textbf{Privacy.} PhotoMind operates on personal data (receipts, food photos, behavioral patterns). All training data stays local; no query patterns are transmitted externally. The RL system learns from query-outcome patterns that may encode purchasing habits, dietary choices, and location information.

\textbf{Silent failure prevention.} The DQN's reward matrix penalizes silent failures most severely ($-1.0$). This prioritizes user safety over coverage---it is better to decline an answerable query than to confidently return wrong information about personal financial data.

\textbf{Routing bias.} The bandit's reward signal derives from ground-truth labels assigned by the photo owner. Learned policies reflect the labeler's intent, not objective truth. Different users' labeling conventions would produce different routing policies.

\textbf{Explainability.} The DQN logs the full state vector and selected action for each query, enabling post-hoc analysis. Future work should surface per-component attribution directly in user-facing responses.

\textbf{Consent and transparency.} Any deployment should inform users that the system improves through query feedback. The current implementation stores no user-identifiable data beyond the local knowledge base.

\section{Conclusion}
\label{sec:conclusion}

PhotoMind demonstrates a complete, production-motivated agentic system with four specialized agents, two distinct workflows, five tools (three built-in + two custom), a 56-query evaluation harness, and an RL extension that eliminates silent failures.

The system integrates two RL approaches---contextual bandits (Thompson Sampling) for query routing across four search strategies and a multi-step DQN for confidence calibration---into a CrewAI multi-agent architecture. The DQN's non-terminal requery action enables multi-step episodes with structurally relevant discounting. Both components train offline via deterministic simulation at zero API cost.

A 7-configuration ablation study across 5 random seeds yields three principal findings:

\begin{enumerate}
  \item The DQN is the primary contributor to silent failure elimination and decline accuracy improvement.
  \item Thompson Sampling complements the DQN on ambiguous queries, while UCB's exploration variance compounds with DQN error modes---a novel interaction-effect analysis for modular RL.
  \item The \textbf{Recommended (Rule+DQN)} configuration offers the best risk-adjusted performance: 76.8\% routing accuracy (identical to baseline), 90.9\% decline accuracy, and 0.4\% silent failures (from 1.8\% baseline).
\end{enumerate}
